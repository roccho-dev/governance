# Proposal: hq terminal proof gate closure

## Goal

Accept hq terminal autocomplete proof evidence as a governance promotion candidate only after hq closes the full proof loop.

## Scope to meta

P10 saleable high-value company -> P9 high-margin software asset -> P8 reusable quality gate -> P7 JSONL-driven input factory -> P0 hq terminal proof gate closure.

## Repo responsibility

governance owns the rule for when hq local evidence may be treated as a shared gate. hq owns the implementation and evidence production.

## Required hq evidence

- engine-backed terminal autocomplete output, not static UI copy.
- actual CLI stdout generated by `python -m hq demo-autocomplete`.
- PNG generated from that stdout in CI.
- strict stdout drift guard.
- scripted REPL or non-interactive acceptance E2E from input buffer to final JSONL row.
- in-repo permanent proof manifest or snapshot that survives expired Actions artifacts.
- practical business-work-item fixture, not only a toy schema.

## Promotion gates

| gate | failure meaning |
|---|---|
| hq-terminal-engine-backed | preview is detached from real hq engine output |
| hq-terminal-stdout-snapshot | review surface changed without explicit approval |
| hq-terminal-png-artifact | visual artifact cannot be regenerated from CLI stdout |
| hq-terminal-repl-e2e | accept path is not proven from input to final JSONL row |
| hq-terminal-business-fixture | proof only covers a toy schema |
| hq-terminal-permanent-proof | proof disappears when CI artifacts expire |

## Close when

- hq finishing PR points to all required evidence.
- every promotion gate has an observable failure meaning.
- governance does not require inspecting transient artifacts only.
- hq evidence can be re-run from the repository.

## Do not close when

- hq still needs another terminal proof PR.
- proof is only a demo stdout without REPL or acceptance E2E.
- proof depends only on expiring GitHub Actions artifacts.
- fixture does not represent practical business input work.
