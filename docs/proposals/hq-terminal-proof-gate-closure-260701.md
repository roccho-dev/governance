# Proposal: hq terminal proof gate closure

## Goal

Define the governance-side receiver for hq terminal autocomplete proof evidence without treating hq as globally gated yet.

## Scope to meta

P10 saleable high-value company -> P9 high-margin software asset -> P8 reusable quality gate -> P7 JSONL-driven input factory -> P0 hq terminal proof gate receiver.

## Repo responsibility

governance owns the promotion rule and observable failure meanings. hq owns implementation, fixtures, commands, and evidence production.

## Required hq evidence before promotion

- engine-backed terminal autocomplete output, not static UI copy.
- actual CLI stdout generated by `python -m hq demo-autocomplete`.
- PNG generated from that stdout in CI.
- strict stdout drift guard.
- scripted REPL or non-interactive acceptance E2E from input buffer to final JSONL row.
- in-repo permanent proof manifest or snapshot that survives expired Actions artifacts.
- practical business-work-item fixture, not only a toy schema.

## Promotion gates

| gate | failure meaning |
|---|---|
| hq-terminal-engine-backed | preview is detached from real hq engine output |
| hq-terminal-stdout-snapshot | review surface changed without explicit approval |
| hq-terminal-png-artifact | visual artifact cannot be regenerated from CLI stdout |
| hq-terminal-repl-e2e | accept path is not proven from input to final JSONL row |
| hq-terminal-business-fixture | proof only covers a toy schema |
| hq-terminal-permanent-proof | proof disappears when CI artifacts expire |

## Receiver close condition

This governance PR closes when the receiver contract is explicit enough for hq #8 to target. It does not claim hq evidence has already passed all gates.

## Close when

- every promotion gate has an observable failure meaning.
- required hq evidence is named in stable terms.
- governance does not require inspecting transient artifacts only.
- the PR clearly says promotion remains blocked until hq provides the evidence.

## Do not close when

- gate names are only labels without failure meaning.
- the receiver claims hq is promoted before hq evidence exists.
- proof can depend only on expiring GitHub Actions artifacts.
- hq implementation responsibility leaks into governance.

## Status

Gate receiver for hq #8. This PR does not implement hq code and does not promote hq evidence yet.
